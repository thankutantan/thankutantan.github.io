Hexo-Next+GitHub搭建个人博客&个性化配置

发表于 2018-09-09 | 分类于 教程 | 本文总阅读量 次
此文章用来记录博主的建站历程，以便备用，或者供人参考。

前言

此篇文章主要针对Hexo-Next主题下的Mist风格，如果有出入，请酌情参考。
文章最后会总结一下修改心得，授人以鱼不如授人以渔！

开始

如果你想深入了解或者想要更多细节的话请观看官方文档
以下可能需要VPN才能访问
hexo
hexo-next

首先，我们明确一下建立博客需要的要素：

  • 一个成熟的博客模版（最好能提供各种风格，并且让程序员修改风格时候方便明了，不需要了解整个前端架构）
  • 一个服务器，能够让我们将博客部署上去以便供人访问

那么有没有什么方法能够解决以上这两个问题呢？
于是乎，Hexo+GitHub这个方法出现了！也就是你现在看到的博客采用的方法。

前期准备

首先，你需要安装

node.js
git

其中node.js是hexo运行需要的环境，git是你部署博客时的需要。
接下来我们准备安装hexo。

打开浏览器输入localhost:4000查看
如果你有看到Hexo几个大字说明安装Hexo成功

如果你不喜欢Hexo原生主题的话，就可以继续下载不同的主题，这里推荐Next

再打开[your-hexo-site]/_config.yml（Hexo配置文件）

这步目的就是将主题改为next

主题修改

这里主要介绍一下一些不常见的修改，基础部分请参照官方文档。

设定自定义首页

这里的首页指登录你域名后展示的页面，正常情况下就是你的博客主页。
但是如果你想用自定义的页面作为首页，再通过首页进入博客呢？
这里继续打开Hexo配置文件[your-hexo-site]/_config.yml

大家看一下英文注释应该就可以了解到这边是运行hexo-g时生成首页的配置。
这里着重介绍一下我们要改的path。

path：即生成首页路径，默认首页在[your-hexo-site]/public/index.html中。将其改成’blog’后，就会生成在[your-hexo-site]/public/blog/index.html
